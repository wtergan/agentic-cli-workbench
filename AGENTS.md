# Agentic CLI Workbench Instructions

This repo is the public-facing showcase project for the user's agentic CLI
workbench. Treat the private dotfiles repo as the source library and this repo
as a curated public subset.

## Workflow

- Use `.vault/PLAN.md` as the project index.
- Execute feature plans from `.vault/plans/` in dependency order.
- Use `.vault/goals/goal-agentic-cli-workbench-2026-05-28/` for native `/goal`
  continuation and handoff.
- Do not publish private overlays, host snapshots, raw screenshots, live agent
  config, auth state, account IDs, emails, or local project history.
- Ask before creating or pushing a GitHub remote.

## Commit Convention

- Make commits atomic.
- Use uppercase type prefixes, for example `FEAT:`, `FIX:`, `DOCS:`, `TEST:`,
  `CHORE:`, `REFACTOR:`, or `SECURITY:`.
- Keep the subject to one main line.
- If a body is useful, include at most four information-dense one-line bullets.
